## 2024-05-18 - Pre-calculating Categorical Hierarchies and Hoisting ML State
**Learning:** In PyTorch inference loops (especially when processing video frames or large batches), resolving device allocation (`next(model.parameters()).device`) and initializing transform pipelines per image creates significant CPU bottlenecks and PyTorch/Python bridge overhead. Additionally, tie-breaking heuristics that rely on dictionary lookups (`HIERARCHY_MAP.get()`) inside key functions for `max()` are slow.
**Action:** Always pre-calculate fallback values for categorical hierarchies into an indexed array and use `max(range(...), key=...)` for tuple comparison. For batched or sequential ML inference, hoist invariant operations like device resolution and transform instantiation completely outside the loop and pass them dynamically as keyword arguments.
