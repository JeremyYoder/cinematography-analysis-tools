## 2024-04-15 - Fast Array Hierarchy Lookup & Transformer Hoisting
**Learning:** PyTorch softmax outputs and list comprehensions coupled with dictionary `.get` tie-breakers in Python are a massive bottleneck. Standard `max()` loops with `.get()` on every element take >0.5s per 100k loops. Pre-calculating hierarchy negative indices into an array and using an index map directly is over 2.5x faster. Furthermore, initializing PyTorch inference transforms (`torchvision.transforms.Compose`) and looking up a model's `.device` inside an iterative directory scanning loop dominates the runtime.
**Action:** When calculating argmax with custom tie-breakers over small categorical spaces, map standard outputs and lookup dictionary results ahead of time. When processing images from a directory loop, ALWAYS instantiate `device` and transform objects entirely outside of the `for` iteration to avoid severe performance degradation.
