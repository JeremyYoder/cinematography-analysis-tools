## 2024-05-18 - Avoid Implicit Matplotlib State in FastAI Loops
**Learning:** `fastai.vision.Image.show()` implicitly creates a new matplotlib figure when called without arguments, leading to severe memory leaks and performance degradation during large batch processing loops (e.g., generating heatmaps). Furthermore, when executing heavy PyTorch model initialization via CLI scripts, validating input arguments (like empty dataset directories) *before* initialization significantly reduces cold-start overhead and prevents unnecessary resource allocation.
**Action:** Always instantiate `Figure` and `Axes` explicitly outside loops (`fig, ax = plt.subplots(...)`), pass them directly to plotting/show functions (`img.show(ax)`), and reuse them via `ax.clear()`. Move expensive ML model initializations in CLI scripts to occur *after* all input validation checks.
