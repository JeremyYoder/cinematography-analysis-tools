## 2024-05-24 - Fix arbitrary file deletion and race conditions in get-heatmaps.py
**Vulnerability:** The script hardcoded `train` and `train/img` directories within user-provided paths to appease FastAI's `ImageDataBunch.from_folder` requirement. It also manually moved user files into these directories, and eventually deleted the hardcoded directory with `shutil.rmtree()`. This risked data loss (if the user already had a `train` directory), race conditions, and potentially arbitrary file deletion.
**Learning:** Reorganizing user files to satisfy framework requirements (like FastAI's strict directory structures) can introduce severe data loss risks and race conditions. Hardcoded temporary directory names must never be used in user directories. Furthermore, complex logic involving moving files should always be wrapped in a deterministic fallback (like `try...finally`) to ensure state restoration upon failure.
**Prevention:** Always use dynamically generated, secure temporary directories (e.g., `tempfile.mkdtemp`) instead of hardcoding names like "train" or "temp". Always wrap file manipulation logic in `try...finally` blocks so that user files are securely restored even if an exception occurs during processing.
