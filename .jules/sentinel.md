## 2024-05-24 - Data Loss Vulnerability in Hardcoded Directories
**Vulnerability:** `get-heatmaps.py` hardcoded a temporary dummy directory named `train` directly inside the user-supplied `path_img` directory. It later used `shutil.rmtree` to indiscriminately delete this directory upon completion.
**Learning:** Hardcoding temporary directory names inside user directories is highly dangerous, especially if it uses a common name like `train`. If a user coincidentally had a directory named `train` inside `path_img` containing their actual data, this script would irreversibly delete it without warning. Additionally, CLI applications do not require 'path traversal' input validation for local filesystem access.
**Prevention:** When a script temporarily requires a dummy directory structure (e.g., for `fastai` `ImageDataBunch` validation) that needs to be deleted later, dynamically provision a secure temporary directory using Python's `tempfile.mkdtemp` (which defaults to safe `0o700` permissions) outside or securely inside the target directory, and use its randomly generated name.
