## 2024-05-16 - Prevent Arbitrary File Deletion & Data Loss in Temporary Directory Operations
**Vulnerability:** The script `get-heatmaps.py` hardcoded a temporary directory name (`train`) within the user-provided image path (`path_img`), moved user files into it, and blindly deleted it (`rmtree(path_img/'train')`) at the end. If the user already had a directory named `train` at that location, their data would be irreversibly deleted. Also, if an exception occurred during processing, files would be left stranded in the temporary directory and not moved back, leading to perceived data loss. (CWE-377 / CWE-379)
**Learning:** Hardcoding temporary directory names in user-controlled paths without checking for existence or using proper temporary directory provisioning libraries is extremely dangerous and can lead to arbitrary data destruction. Additionally, file manipulation operations within a script flow require robust error handling to guarantee state restoration.
**Prevention:**
1. Always use standard libraries like `tempfile.mkdtemp()` to provision dynamically named, secure temporary directories.
2. When moving user files for processing, wrap the logic in a `try...finally` block. Ensure the `finally` block handles moving the files back to their original location and cleaning up the temporary directory, guaranteeing safe state restoration regardless of execution success or failure.
