## 2025-02-21 - FastAPI File Upload DoS and Information Leakage
**Vulnerability:** The FastAPI endpoints `/predict/image` and `/predict/colors` were vulnerable to DoS via unrestricted payload sizes since they read the entire file into memory using `await file.read()`. Additionally, raw exceptions were leaked via `detail=str(e)`, exposing internal architecture/file paths.
**Learning:** In FastAPI, checking `file.size` from `UploadFile` is essential before calling `await file.read()` to prevent memory exhaustion, as `file` content is held in memory or spooled dynamically without global limits by default. Also, any `.egg-info` generated files during `pip install -e .` must be discarded during security fixes unless modifying the root source truth (`pyproject.toml`).
**Prevention:** Always check `file.size` when interacting with `UploadFile` and implement an explicit size limit before loading it into memory. Implement generic HTTP error responses (500) that intentionally scrub `Exception` strings before returning them to clients.
